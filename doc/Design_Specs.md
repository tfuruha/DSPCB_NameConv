# プロジェクト設計書：PCB製造データ・リネーマー (DSPCB_Renamer)

## 1. 目的
電子CADから出力されるデフォルトのファイル名（接尾辞形式）を、基板製造サービス（FusionPCB, JLCPCBなど）で一般的に使用される拡張子形式に一括変換し、製造依頼時の負担を軽減する。

## 2. システム要件
- **OS**: Windows 11
- **言語**: Python 3.x
- **GUIライブラリ**: tkinter (Python標準)
- **依存関係**: 追加ライブラリなし (標準ライブラリのみ)

## 3. 機能仕様
### 3.1 プロジェクト名の抽出
選択されたファイルパスのファイル名から、セパレータ ` - ` (半角スペース + ハイフン + 半角スペース) を基準に分割を行い、最初の要素を `プロジェクト名` として取得する。

### 3.2 ファイル名称変換マップ
プロジェクト名を `[P]` とした場合の変換ルール：

| 変換元 (Source) | 変換後 (Destination) |
| :--- | :--- |
| `[P] - Top Copper.gbr` | `[P].gtl` |
| `[P] - Top Resist.gbr` | `[P].gts` |
| `[P] - Top Silkscreen.gbr` | `[P].gto` |
| `[P] - Bottom Copper.gbr` | `[P].gbl` |
| `[P] - Bottom Resist.gbr` | `[P].gbs` |
| `[P] - Bottom Silkscreen.gbr` | `[P].gbo` |
| `[P] - Mechanical.gbr` | `[P].gko` |
| `[P] - NC Drill Data - [Through Hole].drl` | `[P].txt` |

### 3.3 バリデーションとエラー処理
1. **ファイル特定失敗**: ` - ` が含まれないファイルを選択した場合、エラーを表示。
2. **ファイル不足チェック**: 上記8つのファイルのうち1つでも欠落している場合、不足しているファイル名を一覧表示して処理を中断。
3. **出力先フォルダの作成**: プロジェクト名と同名のサブフォルダを、元のファイルと同じ階層に作成する。既に存在する場合はそれを利用する。
4. **上書き確認**: 作成されたフォルダ内に変換後の名前のファイルが既に存在する場合、`messagebox.askyesno` にてユーザーに確認。
5. **処理後アクション**: リネーム（移動）完了後、作成されたプロジェクトフォルダをファイルエクスプローラーで開く。

## 4. UI設計
