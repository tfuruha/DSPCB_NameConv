# 計画ワークフロー: 製造データファイル整理機能の追加

## 1. 目的
製造データのZIP化作業を効率化するため、リネーム実行時に「プロジェクト名」のサブフォルダを自動作成し、ファイル名変換後のファイル群をそのフォルダ内にまとめて移動・保存する。

## 2. 分析（現状分析）
- **現状**: `DSPCB_Renamer.py` は、選択されたファイルと同じディレクトリ内でファイル名を変更している。変換後のファイルは他のファイル（例えば回路図ファイルやその他の出力ファイル）と同じ階層に混在したままになるため、ZIP化する際に手動でファイルを選択してまとめる必要がある。
- **理想**: リネーム処理時に元のディレクトリ内に `[プロジェクト名]` のサブフォルダを作成し、変換後の8つのファイルをそのフォルダ内に配置する。これにより、ユーザーはそのフォルダを対象にZIP圧縮を行うだけで済むようになり、作業負担が軽減される。

## 3. 提案される変更

### 対象ファイル
- `DSPCB_Renamer.py` (メインロジックの修正)
- `doc/Design_Specs.md` (設計書の仕様更新)

### 具体的な変更内容
1. **出力先ディレクトリの動的生成** (`DSPCB_Renamer.py`)
   - `process_conversion` メソッド内でプロジェクト名を抽出した後、保存先となるサブフォルダのパス (`directory / project_name`) を定義する。

2. **保存先パスの変更とフォルダ作成** (`DSPCB_Renamer.py`)
   - ファイルの存在チェック（`src_path`）は元のディレクトリで行う。
   - 変換後のパス（`dest_path`）を、新しく作成したサブフォルダ内に向ける (`target_dir / dest_name`)。
   - 変換実行直前にサブフォルダが存在しない場合は作成する (`target_dir.mkdir(exist_ok=True)`)。

3. **上書き確認・エクスプローラー起動ロジックの調整** (`DSPCB_Renamer.py`)
   - 上書き確認は、新しいサブフォルダ内の `dest_path` に対して行う。
   - 処理完了後に起動するエクスプローラーの対象パスを調整する（下記「質問事項」の結果に基づく）。

4. **設計書の更新** (`doc/Design_Specs.md`)
   - 新しい仕様である「プロジェクト名サブフォルダの作成」と「ファイルの移動」について追記・修正。

### 検証計画
- `create_test_files.py` を使用してテストファイル群を生成。
- ツールを起動し、テストファイルを選択して実行。
- 元のディレクトリ内に `[プロジェクト名]` フォルダが作成されていることを確認。
- そのフォルダ内に、正しい拡張子に変更された8つのファイルが存在することを確認。
- 元のディレクトリから変更前のファイルがなくなっている（移動されている）ことを確認。
- 上書き実行時、およびフォルダが既に存在する場合の挙動を確認。

## 4. リスク評価
- パーミッション不足等でフォルダが作成できない場合のエラーハンドリングが必要ですが、既存の `try-except` 構文で大半はカバー可能です。念のため、作成前のエラーメッセージ強化を検討します。
- 元ファイルと別ドライブへの移動になることはありません（同階層のサブフォルダであるため）が、Windows上での「ファイル移動+リネーム」を安全に行うため、`Path.rename()` の動作確認を徹底します。

## 5. 質問事項（不明点）
実装を進めるにあたり、以下の点について方針を教えてください。

1. **元のファイルの扱い:**
   今回は「リネーム」処理の延長として、元のファイルを新しいフォルダへ「移動（ファイル名変更と同時に元の場所からは削除）」する想定で良いでしょうか？（元の場所に変換前のコピーを残す必要はありませんか？）
   A1: 元の場所には変換前のコピーを残さず、新しいフォルダへ移動する。
2. **処理完了後のエクスプローラー表示:**
   現在、処理完了後に変換されたファイルがあるフォルダを開いています。今回は「親フォルダ（プロジェクトフォルダが作成された場所）」を開くのと、「作成されたプロジェクトフォルダの中」を開くのと、どちらがZip化作業において便利でしょうか？
   （Zip化作業を直ちに行うのであれば、親フォルダを開いたほうが、作成されたフォルダを右クリックしてすぐZip圧縮できるため便利かもしれません）
   A2: 製造データの確認を外部ツールで確認するので、作成されたプロジェクトフォルダの中を開く。
3. **既存のプロジェクトフォルダがある場合:**
   すでに同名のサブフォルダが存在していた場合、そのフォルダをそのまま利用して中にファイルを出力（必要に応じて上書き確認）する挙動でよろしいでしょうか？
   A3: そのフォルダをそのまま利用して中にファイルを出力する（必要に応じて上書き確認）。
